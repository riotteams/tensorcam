HiResCAM

HiResCAM, an explanation method in the CAM family that is faithful to the model, solves a limitation of Grad-CAM which is due to its gradient averaging step, Grad-CAM is not guaranteed to reflect locations the model used for prediction, and therefore can produce misleading explanations. The purpose of HiResCAM is to reflect the model’s computations; therefore, when the gradients indicate that some elements of the feature map should be scaled or have their sign inverted, HiResCAM performs these operations and is guaranteed to highlight all locations within the image that increase the class score. In contrast, Grad-CAM blurs the effect of the gradients across each feature map. HiResCAM can be applied to 2D, 3D, or n-D CNNs.

the HiResCAM explanation for a model with a CAM architecture is:

A m H i R e s C A M = 1 D 1 D 2 ∑ f = 1 F w m f A f

the gradient ∂ S m ∂ A ( f ) :

∂ S m ∂ A ( f ) = 1 D 1 D 2 w m f
How do I use this technique on an image?

To load a HiResCAM class:

The constractor have three parameters:

  • model : Input Model To Compute The HiResCAM
  • layer_name : Target Layer Or Its Name (Optional Parameter)
  • Class-index : Feature Class Index Of The Object That We Need To Find On The Image

We will use Resnet50 model for test

from tensorcam import HiResCAM
from tensorflow.keras.applications import ResNet50

class_index = {'tiger_cat': 282, 'beagle': 162}
cam = HiResCAM(model, class_index['beagle'] ,layer_name='conv5_block3_out')
The HiResCAM class have 3 methods :

(get_image) : that takes the original image and retern the image as a numpy 2d array and the origenal image self.
(compute_cam_features) : that takes a 2d numpy array and return the heatmap.
(plot_images) : that takes the original image , the heatmap and return the image with the heatmap.

image, orig = HiResCAM.get_image('sample.jpg')
cam_features = cam.compute_cam_features(image)
cam.plot_images(orig, cam_features)
